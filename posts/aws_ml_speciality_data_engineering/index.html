This domain requires understanding of creating data repositories for machine learning, identification and implementation of data ingestion solution, and identification and implementation of a data transformation solution.

S3

  • foundational for machine learning projects since it is a cost effective solution for datasets storage
  • object based storage, bucket name need to be globally unique, however they are unique to regions
  • key is the full path of the file and even though it looks like there is a folder based heirarchy, that is not how it works
  • you can have a very long file name, in the sense that the path (key) can be very long
  • max size is 5 TB
  • object tags can be added, helpful with classificaiton and security lifecycle, these are key value pairs
  • decoupling of compute and storage side
  • perfect use case of data lake, since it can store various formats of data (object storage)
  • it is possible to partition the storage, which is helpful (speedy) when querying via athena. Kinesis partitions the data automatically.
  • 11 9’s durability

Storage classes

  • S3, Standard or General Purpose: for frequently accessed data
  • S3, Infrequent Access: lower cost than standard, for data accessed monthly, and requires milliseconds retrival, but there is a cost associated with retrival
  • S3, Infrequent Access, One Zone, good for secondary copies of backup, or data that can be recreated, infrequent access for cost saving
  • S3, Glacier Instant Retrival, price per storage + price per retrival, can access within milliseconds, for low cost storage for long-lived data
  • S3, Glacier Flexible Retrival, expedited: 1-5 mins, standard: 3-5 hrs, bulk: 5-12 hrs (free), for long term low cost storage for backups and archives with different retrival options
  • S3, Glacier Deep Archive: lowest cost, 180 days of minimum storage, for rarely accessed archive data
  • S3, Intelligent Tiering: move objects between tiers with monthly monitoring and auto-tiering fee
  • It is possible to move objects between these storage classes

Lifecycle rules

  • Transitioning objects between classes is possible
  • Transition Actions can be used to configure objects to transition to another storage class
  • Transition Actions can also be used for expiration, incomplete multi part uploads etc.
  • Rules can be applied to buckets, specific paths of the project or also to tags
  • Amazon S3 analytics works exclusively on S3 standard, and S3 IA, and provides analytics on usage

Encryption

  • Server Side encryption managed by AWS (AES 256)
  • Server Side encryption managed by KMS
  • Server Side encryption managed by Consumer (not relevant for ml exam)
  • Client side encryption (not relevant for ml exam)

Security Policy

  • Identity based or resource based access
  • default encryption is possible

Misc

  • VPC can be used to stay within private network
  • logging audit is possible (can be sent to another S3 or using cloudtrail)
  • Tags can be used for security

Kinesis

  • Kinesis is a managed alternative to Kafka
  • “real time” big data
  • Data is replicated to at least 3 AZ
  • Great for streaming processing frameworks
  • Great for logs, metrics, events, IoT, clickstreams, etc

Kinesis Streams

  • low latency streaming ingest at scale
  • streams are divided into shards and partitions
  • the maximum throughput of a single shard 1 mb/secondss or 1000 messages/seconds
  • data retention: 24 hours by default, data retention. it can go up to 365 days. This is useful for reprocessing/replaying data
  • immuatble, 1 mb in size
  • Provisioned mode: choose number of shards and scale manually or using an API
  • each shard gets 1mb/s in, 2mb/s out
  • on demand mode: each capacity provisioned is 4mb/s
  • if you can plan capacity, use provisioned. however, use on demand if capacity is unknown
  • real time streaming application, but you provision in advance
  • custom code for producer or consumer is possible
  • real time (200 ms latency, possible all the way up to 70ms)
  • automatic scaling with on-demand mode
  • multi consumers is possible from one source

Kinesis Analytics

  • perform real-time analytics on streams using SQL or Apache Flink
  • select columns, continious metric generation, responsive analytics, etc.
  • serverless, scales automatically, pay for resouces consumed but expensive
  • schema discovery
  • lambda can be used for preprocessing
  • Two machine learning algorithms
  • Random cut forest for anomaly detection on numeric columns in a stream, uses recent data to compute the model
  • hotspots, finds dense clusters, uses all data

Kinesis Firehose

  • load streams into S3, Redshift, ElasticSearch, Splunk and several others
  • store data in target application
  • most common is firehose reading from kinesis streams
  • near realtime service because it batch writes
  • data desitination can be s3, redshift, elastisearch, splunk, new relic, or http endpoint
  • firehose is a fully managed service
  • 60 seconds latency minimum for non full batches
  • data ingestion into redshift, s3, elasticsearch, splunk
  • automatic scaling
  • conversions from csv/json to parquet and orc and requires the use of glue
  • and transformation through lambda csv to json is possible
  • compression is possible
  • automates scaling
  • no data storage
  • no replay capability
  • it is a serverless transformation tool

Kinesis Video Streams

  • meant for streaming video in real-time
  • producers: security camera, body-worn cam, aws deeplens, radar data, camera
  • one producer per video stream
  • video playback capability
  • Sagemaker, rekognition video, 1 hour to 10 years
  • Checkpointing via dynamodb, frames to sagemaker for ml inference, publish to stream, lambda can be used for notification

Glue

Glue Data Catalog and Glue Data Crawlers

  • Automated Schema Inference
  • Schemas are versioned
  • integration with Athena and Redshift Spectrum
  • Glue crawlers help build the Catalog
  • crawlers can infer different data types
  • and for different services S3, Redshift, RDS etc.
  • Glue will also extract the partitions, this is helpful for query optimization

Glue ETL

  • Transform data, clean data, enrich data before doing analysis
  • generate etl code in python or scala, you can modify the code
  • can provide your own spark or pyspark scripts
  • target can be s3, jdbc or in glue data catalog
  • fully managed, cost effective, pay only for the resources consumed
  • jobs are run on a serverless Spark platform
  • glue scheduler to schedule the jobs
  • glue triggers to automate job runs based on events
  • transformations can be bundled (drop, filter, join, map)
  • machine learning transformation (find matches, duplicates even when data do not match exactly, dedup with glue is possible)
  • any apache spark transformation is possible, and changing in format is possible.
  • multiple ways to create glue jobs including visual editors, python notebooks, python script, etc.

Redshift

  • OLAP
  • uses SQL to analyze structured and semi-structured data across data warehouses, operational databases, and data lakes
  • Spectrum can directly read from S3
  • columner

RDS

  • OLTP
  • must provision servers in advance

DynamoDB

  • NoSQL
  • serverless
  • useful to store ml model (or checkpoints)

OpenSearch

  • Previously ElasticSearch
  • Indexing of data
  • Search amongst data points
  • Clickstrems analytics

AWS Data Pipeline

  • ETL service
  • S3, RDS, DynamoDB, Redshift, EMR
  • manages task dependencies
  • data sources can be on premise
  • runs on EC2 but managed by EC2
  • Orchestration service
  • Glue is managed, serverless, spark focused, etl focused, has catalog
  • Data Pipeline is orchestation tool, and can do more

AWS Batch

  • Run batch jobs
  • as Docker images
  • fully serverless
  • batch can be scheduled using cloudwatch, step functions
  • not just for ETL but absolutely anything at all]
  • EC2
  • For any non-ETL batch is usually better than glue

AWS DMS

  • Data Migration Service
  • Quickly and securely migrate databases to AWS
  • Supports homogeneous migrations and heterogeneous migrations
  • continuous data replication

Step Functions

  • use to design workflow
  • advanced error handling and retry mechanism
  • audit of history of workflow
  • wait for some time
  • maximum execution time of 1 year
  • can be used to train a ml model
  • can also be used to tune a ml model

Misc

  • AWS DataSync: for data migrations from on-premises to AWS storage services
  • MQTT: IOT protocol, Standard messaging protocol
